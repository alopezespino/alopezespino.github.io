Secure Remote Access: A Guide to SSH Keys for Servers and HPC Clusters

by Alejandra López Espino | Mar 8 2024 · 3 min read
#SSHkeys #remote-access #HPC #cluster #RoarCollab #tutorial #server

What are SSH keys

SSH keys are a pair of cryptographic keys that enable secure, password-less remote access to servers and other systems.

Benefits

Leveraging SSH authentication can significantly enhance your research workflow with High Performance Computing (HPC) resources. It allows for seamless file exchanges between your local computer and the server directly from the terminal, eliminating the need to repeatedly input your account password. Moreover, SSH authentication enables you to work on your code locally while executing it remotely using, for instance, VS Code. I explain how to configure VS Code for this purpose in another post. However, please be aware that for servers employing two-factor authentication, this secondary verification step will remain necessary. Specifically, for the Roar Collab server, this step is required just once every 24 hours.

External validity

These instructions are specifically designed for Roar Collab, Penn States’s High Performance Computing (HPC) Cluster, but they are universally applicable to any server equipped with SSH and SCP capabilities. Whether you’re working with a different HPC cluster or a standard Linux server, the steps outlined here will guide you in securely configuring access with an SSH key. Throughout these instructions, you’ll see placeholders like <psu-id> and submit.hpc.psu.edu. Simply replace <psu-id> with your unique user ID or username, and submit.hpc.psu.edu with the address of your specific server or HPC cluster.

Step 1: Setting Up Terminal Shortcuts

I’ve created aliases and functions for convenience. Add these to your .bashrc file:

  • Starting the SSH Agent
$ alias startssh="eval `ssh-agent -s`"

  • Transferring Files from the Terminal

    • From Local to Rcollab:

function take() {
        scp -r $1 <psuid>@submit.hpc.psu.edu:$2
}

This function, take(), is designed to securely copy a directory specified by the first parameter to a remote server at submit.hpc.psu.edu. The path of the directory on the remote server is specified by the second parameter.

  • From Rcollab to Local:

function bring() {
   scp -r <psuid>@submit.hpc.psu.edu:$1 $2
}

Similarly, the bring() function is designed to securely copy a directory specified by the first parameter from a remote server at submit.hpc.psu.edu to the local machine. The path of the directory on the remote server is specified by the first parameter, and the destination path on the local machine is specified by the second parameter.

Step 2: Generating SSH Key

Generate your SSH key with the following command:


ssh-keygen

When prompted, choose a name for your SSH key. For passphrase, simply press Enter. press Enter once again when asked for the passphrase the second time.

Step 3: Copying Key to Server

Use the take function to copy your SSH key to the server:


take ~/.ssh/r2rc /storage/home//.ssh/
ssh-copy-id -i ~/.ssh/r2rc @submit.hpc.psu.edu

Step 4: Configuring the Connection

Add the following lines to your ~/.ssh/config file to configure the connection:


Host rcollab submit.hpc.psu.edu
  HostName submit.hpc.psu.edu
  IdentityFile /storage/home//.ssh/r2rc
  User 
  ForwardX11 yes
  ForwardX11Trusted yes

Troubleshooting

If the connection still doesn’t work, start the SSH agent on the server:


startssh

Then, add the key:


ssh-add r2rc
